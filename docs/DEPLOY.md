# Безкоштовний деплой Argus

Повна вартість: **$0** (Firefox AMO + Render free tier + Groq free tier).
Якщо захочеш ще й Chrome Web Store — $5 одноразово за акаунт розробника.

Порядок важливий: спершу бекенд (щоб дізнатись його URL), потім політика
конфіденційності, і аж тоді прод-збірка розширення — бо URL бекенду
вшивається у бандл на етапі збірки.

---

## Крок 1. Бекенд на Render ($0)

### 1.1 Створи ключ Groq (безкоштовно)
1. https://console.groq.com → зареєструйся (без картки).
2. **API Keys** → **Create API Key** → скопіюй (показується один раз).
3. Там же в **Models** звір актуальні назви моделей — вони змінюються.
   У `render.yaml` за замовчуванням стоять `llama-3.3-70b-versatile` (чат) і
   `llama-3.1-8b-instant` (пояснення).

### 1.2 (Опційно) Ключ Etherscan
Без нього історія й аналітика **EVM-мереж** будуть порожні (Bitcoin/Solana
працюють без ключа). https://etherscan.io/apis → free tier, один ключ на всі
5 EVM-мереж.

### 1.3 Деплой
1. https://render.com → **Sign up with GitHub** (картка не потрібна).
2. **New** → **Blueprint** → обери репозиторій `San4o-0/AIwebWallet`.
3. Render підхопить `render.yaml` і покаже сервіс `argus-api`.
4. Він попросить ввести секрети (позначені `sync: false`):
   - `OPENAI_API_KEY` — ключ **Groq** (так, назва змінної openai — бекенд
     працює з будь-яким OpenAI-сумісним API через `OPENAI_API_BASE`).
   - `ETHERSCAN_API_KEY` — якщо є; інакше лиши порожнім.
5. **Apply** → чекай збірку (перша ~5–10 хв: Rust компілюється).

### 1.4 Перевір
Render дасть URL виду `https://argus-api.onrender.com`.

```bash
curl https://argus-api.onrender.com/health
# {"status":"ok","version":"0.1.0"}

curl -X POST https://argus-api.onrender.com/v1/balances \
  -H 'Content-Type: application/json' \
  -H 'Origin: moz-extension://test' \
  -d '{"addresses":{"evm":["0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045"],"solana":[],"bitcoin":[],"tron":[]}}'
# має повернути реальні баланси
```

### 1.5 Пінг проти засинання (безкоштовно, дуже раджу)
Free tier присипляє сервіс через **15 хв простою**; холодний старт — 30–60 с.
Розширення це переживає (довгий таймаут + екран «сервер прокидається»), але
краще не спати взагалі:

1. https://uptimerobot.com → безкоштовний акаунт.
2. **Add New Monitor** → тип **HTTP(s)**.
3. URL: `https://argus-api.onrender.com/health`
4. Інтервал: **5 хвилин**.

Кореневий `/health` навмисно виведений з-під rate limiting, тож пінг нічого
не з'їдає.

## Крок 2. Політика конфіденційності на GitHub Pages ($0)

Обидва стори вимагають політику за публічним URL.

1. Впиши в `docs/PRIVACY.md` свої дані замість `TODO` (email для зв'язку і
   URL репозиторію) — без робочого контакту стори відхиляють.
2. У GitHub: **Settings** → **Pages** → Source: **Deploy from a branch** →
   branch `main`, folder `/docs` → **Save**.
3. Через ~1 хв політика буде за адресою:
   `https://san4o-0.github.io/AIwebWallet/PRIVACY`

> Ця адреса вже вшита в екран згоди розширення (`PRIVACY_POLICY_URL`).
> Якщо вирішиш хостити інакше — онови константу і перезбери.

## Крок 3. Прод-збірка розширення

```bash
cd extension
VITE_API_BASE_URL=https://argus-api.onrender.com/v1 pnpm build -b firefox
```

Перевірки, які роблять це безпечним:
- Збірка **впаде**, якщо URL не `https://` (захист від cleartext — MITM міг би
  підмінити `chain_id` і комісії, які підписуються).
- URL потрапляє і в код, і в `host_permissions` маніфеста — вони не розійдуться.

Артефакт: `.output/firefox-mv2/`. Спакуй для подачі:

```bash
cd .output/firefox-mv2 && zip -r ../../argus-firefox.zip .
```

## Крок 4. Подача у Firefox AMO ($0)

1. https://addons.mozilla.org/developers/ → увійди (акаунт Mozilla безкоштовний).
2. **Submit a New Add-on** → **On this site** (щоб з'явився в каталозі) або
   **On your own** (self-distribution — теж підписується, але не в каталозі).
3. Завантаж `argus-firefox.zip`.
4. Заповни:
   - **Опис** — обов'язково згадай, що це AI-гаманець, який пояснює транзакції
     (це «єдина мета» розширення; передача даних AI-провайдеру має бути з нею
     узгоджена).
   - **Privacy Policy** — URL із кроку 2.
   - **Категорії**, скриншоти, іконка (вже в збірці).
5. **Source code**: AMO вимагає вихідники, якщо бандл мініфікований або містить
   WASM. У нас і те, і те → додай архів джерел і коротку інструкцію збірки
   (Rust + wasm-pack + pnpm). Без цього рев'ю відхилить.

**Строки:** зазвичай кілька днів. Гаманці Mozilla перевіряє прискіпливіше
(з 2025 у них окремий ризик-скоринг саме для wallet-розширень).

## Крок 5 (опційно). Chrome Web Store ($5)

1. https://chrome.google.com/webstore/devconsole → $5 одноразово.
2. Увімкни **2FA** на акаунті — без неї публікація заблокована.
3. Прод-збірка: `VITE_API_BASE_URL=... pnpm build` (chrome — за замовчуванням).
4. **Data disclosure**: задекларуй збір фінансових даних і передачу третім
   сторонам (AI-провайдер). Політика конфіденційності — обов'язкова.
5. **DSA trader verification** (юр. назва, адреса, телефон) — потрібна лише для
   дистрибуції в **ЄС**. Не хочеш світити дані — оголоси себе «не-трейдером»,
   але тоді розширення не показуватиметься в ЄС.

## Що з обмеженнями безкоштовних тарифів

| Сервіс | Ліміт | Що буде при перевищенні |
|---|---|---|
| Render free | Присипляє через 15 хв; 750 год/міс | Холодний старт (лікується пінгом) |
| Groq free | Обмеження запитів/хв і токенів/день | AI-пояснення тимчасово недоступні → падають на локальні шаблони (гаманець працює) |
| Etherscan free | 5 зап/сек, 100k/день | Історія EVM тимчасово порожня |
| Публічні RPC | «Чесне користування» | Ретраї з backoff уже вбудовані в бекенд |

Усі ці збої **не ламають гаманець** — він деградує чесно, показуючи стан
помилки замість вигаданих даних.
